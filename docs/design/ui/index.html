Design system · reference

hron, with a coat of paint.

hron is a small, sharp idea: schedules you can read. It already does everything cron does and more — multi-week intervals, ordinals, yearly dates, timezones with DST. The engineering is done; this is the design it deserves.

The direction: crisp, fast, quietly technical. One ink, one spark of vermillion, a display face with edges (Space Grotesk) paired with a real code face (JetBrains Mono). The product is a language, so type and the literal cron→hron transform carry the page. Below: the new identity, the palette, and two live screens you can click into.

The mark

A monoline clock — frozen at 9:00, the canonical "every weekday at 9:00". The vertical hand is the vermillion spark; the rest is ink. No tick marks, no toy hands. Reads cleanly from favicon to billboard. Exportable SVGs ship in /assets.

Palette

Mostly ink and paper. Vermillion is the only loud color — it marks the "human" half of every transform. A muted green stands in for the "machine" (cron) side.

What changed

Same content, sharper delivery. Nothing here invents features — it dramatizes what's already true about hron.

Before
  • Generic dev-tool template, blue accent
  • System fonts, flat comparison table
  • Clip-art clock logo with toy-red hands
  • Playground: bare textarea, plain list
After
  • Hero leads with the live cron→hron transform
  • Space Grotesk + JetBrains Mono, vermillion spark
  • Single monoline 9:00 clock — scales to a favicon
  • Playground: cron field breakdown + dated, relative occurrences
